# 무브캐스터즈: 포즈 브레이크

키보드로 전투하고, 결정적인 순간에는 웹캠 앞에서 포즈를 취해 스킬을 발동하는 웹캠 포즈 액션 로그라이트입니다.

평소에는 탑다운 액션 게임처럼 움직이고 피하며 싸웁니다. 스킬 버튼을 누른 순간에만 카메라가 포즈를 읽고, 성공한 포즈는 캐릭터의 강력한 기술로 이어집니다.

## 핵심 특징

- **웹캠 포즈 스킬**: Q/E/R/F로 스킬을 준비한 뒤 손동작과 상반신 포즈로 시전합니다.
- **카메라 AR 효과**: 카메라 미리보기 위에 손목 glow, 제스처 trail, 보스 카운터 burst가 표시됩니다.
- **캐릭터 4명**: 리오, 마루, 네온, 쿠키가 서로 다른 제스처와 전투 스타일을 가집니다.
- **로그라이트 보상 카드**: 웨이브를 돌파하며 카드 보상을 고르고, 태그 기반 시너지를 완성합니다.
- **보스 포즈 브레이크**: 보스의 큰 패턴을 특정 포즈로 막아 카운터합니다.
- **훈련장/튜토리얼/허브**: 포즈 연습, 성장 확인, 설정 변경, 최근 기록 확인이 가능합니다.

## 조작법

| 입력 | 기능 |
| --- | --- |
| WASD / 방향키 | 이동 |
| Space | 대시 |
| Q / E / R / F | 스킬 준비 |
| 웹캠 포즈 | 준비한 스킬 시전 |
| 1 / 2 / 3 | 대체 입력: 노멀 / 그레이트 / 퍼펙트 |
| Tab | 선택한 카드와 현재 빌드 확인 |
| Esc | 일시정지 |
| D 또는 ` | 개발자 디버그 패널 |

## 카메라 안내

카메라를 쓰려면 브라우저 권한 허용이 필요합니다.

1. 게임 안에서 `카메라 켜기`를 누릅니다.
2. 브라우저의 카메라 권한 요청을 허용합니다.
3. 상반신, 양어깨, 양손이 화면에 들어오게 맞춥니다.
4. 카메라 보정에서 슬래시, 크로스 가드, 오픈 암 테스트를 확인합니다.
5. 스킬 준비 후 안내 포즈를 취하면 포즈 캐스트가 발동합니다.

카메라 영상은 저장하지 않습니다. 포즈 인식은 브라우저 런타임에서 처리되며, 설정에서 원본 영상 표시, AR 이펙트 강도, 스켈레톤 표시, 카메라 크기를 조절할 수 있습니다.

## 카메라 없이 테스트하기

카메라가 없거나 권한을 허용하지 않아도 플레이할 수 있습니다.

스킬을 준비한 뒤 아래 키로 포즈 판정을 대신 입력합니다.

| 입력 | 판정 |
| --- | --- |
| 1 | 노멀 |
| 2 | 그레이트 |
| 3 | 퍼펙트 |

튜토리얼, 훈련장, 보스 포즈 브레이크도 같은 방식으로 테스트할 수 있습니다.

## 캐릭터

- **RIO**: 고속 번개 딜러. Slash / Thrust 중심의 빠른 공격과 대시 시너지가 강합니다.
- **MARU**: 방어/반격 탱커. Cross Guard / Palm Push로 보호막과 반격을 활용합니다.
- **NEON**: 표식/함정 암살자. Point / Circle / Focus Triangle로 적을 해킹하고 폭발시킵니다.
- **COOKIE**: 소환/장판 서포터. Circle / Spread / Heart로 슬라임과 젤리 장판을 만듭니다.

## 게임 모드

- **3분 데모**: 짧은 시간 안에 이동, 스킬, 보상 카드, 새 적, 보스 포즈 브레이크를 체험합니다.
- **튜토리얼**: 이동, 대시, 자동 공격, 포즈 캐스트, 보상 카드, 보스 포즈 브레이크를 순서대로 익힙니다.
- **모험**: 루프시티, 놀이공원, 미러 타워, 제로 존을 단계별로 클리어합니다.
- **전투**: 5웨이브 기반 기본 전투 모드입니다.
- **생존**: 제한 시간 동안 몰려오는 적을 버팁니다.
- **보스 러시**: 보스를 연속으로 상대하며 포즈 브레이크 숙련도를 시험합니다.
- **훈련장**: 제스처별 미션과 점수 breakdown을 보며 포즈를 연습합니다.
- **허브**: 캐릭터 성장, 코어, 스킨, 이펙트, 업적, 최근 기록, 난이도와 카메라 설정을 확인합니다.

## 로컬 실행

```bash
npm install
npm run dev
```

개발 서버 기본 주소는 다음과 같습니다.

```text
http://127.0.0.1:5173
```

카메라 권한은 `localhost` 또는 `127.0.0.1` 환경에서 사용할 수 있습니다.

## 빌드

```bash
npm run build
```

빌드 결과물은 `dist/` 폴더에 생성됩니다.

로컬에서 빌드 결과를 확인하려면 다음 명령을 사용합니다.

```bash
npm run preview
```

## GitHub Pages 배포

이 프로젝트는 GitHub Pages 배포를 위해 Vite `base`가 저장소 이름에 맞춰 설정되어 있습니다.

```ts
base: "/web-game-making/"
```

`main` 브랜치에 push하면 `.github/workflows/deploy.yml`이 실행됩니다.

1. 저장소 코드를 체크아웃합니다.
2. Node.js 24를 설정합니다.
3. 의존성을 설치합니다.
4. `npm run build`를 실행합니다.
5. `dist/`를 GitHub Pages artifact로 업로드합니다.
6. GitHub Pages에 배포합니다.

GitHub 저장소의 `Settings > Pages`에서 Source를 `GitHub Actions`로 설정해야 합니다.

배포 후 주소 형식은 다음과 같습니다.

```text
https://<github-user>.github.io/web-game-making/
```

## 브라우저 권장 사항

- 최신 Chrome 또는 Edge 권장
- 카메라 사용은 HTTPS 또는 localhost 환경 필요
- GitHub Pages는 HTTPS로 제공되므로 카메라 권한 요청 가능
- 저사양 환경에서는 설정에서 카메라 AR 강도와 퍼포먼스 모드를 낮추는 것을 권장

## 알려진 이슈

- MediaPipe 모델과 Phaser 번들이 커서 첫 로딩에 시간이 걸릴 수 있습니다.
- 카메라 포즈 인식은 조명, 카메라 각도, 손이 화면 밖으로 나가는 상황에 영향을 받습니다.
- 일부 브라우저는 카메라 권한을 차단하면 다시 허용하기 위해 사이트 설정을 직접 열어야 할 수 있습니다.
- 모바일 브라우저에서는 화면이 좁아 HUD와 카메라 패널이 compact 배치로 보일 수 있습니다.
- 웹캠이 없어도 1/2/3 대체 입력으로 주요 게임 흐름은 테스트할 수 있습니다.
